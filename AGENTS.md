# AGENTS.md

`rice-knowledge-map` で作業するエージェント向けの基本ルールです。

## 1. 目的

このプロジェクトは、実画像マップ型の稲作ナレッジ記録アプリを完成させるためのものです。

テストモックは途中工程です。最終目的は、自分の田んぼで使えるPWAアプリを作ることです。

中心は、作業一覧や日報ではなく、実画像マップです。

## 2. UI判断

参照モック画像をUI判断の基準にします。

文章の要素リストよりも、参照画像の見た目、余白、質感、画面密度、スマホアプリ感を優先してください。

1画面にすべての情報を詰め込む必要はありません。見た目を悪くする要素は、小さくする、下部シートに入れる、タップ後に出す、別画面へ移す、初期表示から外す、のいずれかで扱います。

## 3. 目指す方向

- フルスクリーンに近い実画像マップ
- 薄く軽いヘッダー
- 地図上の田んぼ区画とピン
- 下部ボトムシート
- フローティング操作ボタン
- 写真で記録、音声メモへの強い導線
- 片手で操作しやすいタップ領域
- 余白、角丸、影、半透明、重なりのあるスマホアプリUI

## 4. 避ける方向

- 古い業務システム風
- 管理ダッシュボード風
- 表や一覧が主役の画面
- 白いカードを並べるだけの画面
- 大きすぎるヘッダー
- 多すぎるフィルター
- ラベルだらけの地図
- 天気、予定、同期状態が主役に見える画面

## 5. 技術方針

```text
Frontend : Next.js App Router
Language : TypeScript
UI       : React + Tailwind CSS
Map      : MapLibre GL JS
Base map : 国土地理院 空中写真タイル
Backend  : Supabase Auth / Postgres / Storage / RLS
Deploy   : Vercel Preview
```

UI確認も最終実装もNext.js内で行います。HTMLだけの別モックは原則作りません。

Tailwind CSSを標準とし、`globals.css` は必要最小限にします。

## 6. 現在の最優先

現在の最優先は、メインマップ画面を参照モック画像に近いスマホアプリUIへ引き上げることです。

この品質が合格するまで、DB接続や機能追加へ進まないでください。

## 7. 作業単位

1タスクは1目的に限定します。

UI作業、DB作業、認証作業、保存処理を混ぜないでください。

UIタスクのDONE条件:

- スクリーンショットまたはPreview URLがある
- 参照モック画像と比較している
- 実画像マップが主役に見える
- 変更ファイルが説明されている
- 未確認事項が明記されている
