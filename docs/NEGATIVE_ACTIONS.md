# NEGATIVE_ACTIONS.md

## 1. 目的

この文書は、AIエージェントがユーザー承認なしに実行してはいけない操作を定義します。

## 2. 原則

- 迷ったら実行しない。
- 実行前に目的、影響範囲、戻し方を提示する。
- 承認がない限り実行しない。
- 既存データ、既存設定、権限、本番環境を勝手に変更しない。
- 多分大丈夫で進めない。

## 3. 承認必須操作

### GitHub

- 既存ファイル削除
- mainブランチへの直接push
- PR merge
- force push
- リポジトリ公開設定変更
- branch削除
- 大規模なファイル移動
- 大規模リファクタリング

### Supabase

- migration適用
- 既存テーブル削除
- 既存カラム削除
- 既存データ更新または削除
- RLS policy変更
- RLS無効化
- Storage公開設定変更
- 認証設定変更
- Edge Function deploy

### Vercel

- 本番デプロイ
- 環境変数変更
- ドメイン設定変更
- Production branch変更
- プロジェクト削除

### 外部サービス

- 課金が発生する操作
- 有料プラン変更
- 公開URLの共有範囲変更

## 4. 承認なしで可能な操作

- GitHub repo状態確認
- ファイル内容確認
- docs追加
- tasks追加
- feature/docsブランチ作成
- Issue作成
- PR作成
- PRレビュー
- Supabase状態確認
- テーブル一覧確認
- RLS状態確認
- マイグレーションSQL案作成
- Vercel Preview確認

## 5. 実行前テンプレート

```text
操作名:
目的:
対象:
影響範囲:
変更内容:
戻し方:
実行担当:
承認待ち:
```

## 6. 完了報告テンプレート

```text
実行操作:
実行日時:
実行担当:
対象:
結果:
確認方法:
戻し方:
未確認事項:
```

## 7. 禁止事項

- 未承認の削除
- 未承認の本番反映
- 未承認のDB変更
- 未確認の完了報告
- エラー回避のための仕様無視
- 完了していないタスクのDONE化
