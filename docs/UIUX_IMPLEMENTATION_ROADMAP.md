# UI/UX 実装ロードマップ

最終更新: 2026-06-26

## 0. この文書の目的

この文書は、実画像マップ型 稲作ナレッジ記録アプリを、現場で使えるMVPへ段階的に引き上げるための実装順序です。

現在は「マップ中心の基本操作が形になった」段階です。アプリ全体のUI/UX、初回利用導線、記録導線、使い方、READMEは未完成です。

### 守ること

- 1 PR = 1目的。UI改修、構造整理、DB変更を混ぜない。
- 既存の実機確認済みフローを壊さない。
- UIを先に決め、DBや新機能を増やす前に操作体験を固める。
- 完了は「ビルド成功」だけでは不可。対象フローのPreview確認と、必要なiPhone実機確認を証拠に残す。
- 迷ったら「写真を撮る → 話す → 保存する → 実画像マップに残る」が短くなる方を選ぶ。

### 完了状態

| 状態 | 意味 |
|---|---|
| TODO | 未着手 |
| IN_PROGRESS | 実装または実機確認中 |
| DONE | PRマージ済み。必要な実機確認の証拠あり |
| DEFERRED | MVPでは後回し。再開条件を明記 |

## 1. 現在の基準線

### 完了済み

- マップを中心にした田んぼ選択、場所合わせ、輪郭描画、名前入力、保存
- 1本指描画時のiOSブラウザジェスチャー漏れ防止
- 田んぼ・地点ピン・記録・写真・音声・コメント・対応済み管理の基本機能
- モバイルDrawer / PC SideNav の基本ナビゲーション
- 田んぼ選択ホイールピッカー

### 現時点で壊してはいけない代表フロー

1. `/map` → 田んぼを登録する → 場所を合わせる → 輪郭を描く → 名前入力 → 保存
2. `/map` → 田んぼを選ぶ → ホイールを動かす → 正式選択 → 詳細を見る
3. 田んぼ詳細 → ピン追加 → 保存
4. 写真・音声を含む記録作成 → 保存前確認 → 保存 → 詳細表示
5. 記録詳細 → コメント → 対応済み

## 5. 実装運用

1. このロードマップから1件だけ `IN_PROGRESS` にする。
2. 実装前に、対象画面・対象外・完了条件・実機確認項目をPR本文へ書く。
3. 実装後にVercel Previewで確認する。
4. iPhone固有の操作が含まれる場合は、ユーザー実機確認後にのみマージする。
5. マージ後に `tasks/TASKS.md` の履歴へ証拠を追加し、ロードマップの状態を更新する。
