# REVIEW_CHECKLIST.md

## 共通確認

- 要件定義に反していないか
- 実画像マップ中心か
- 普通の農業日誌に寄っていないか
- 手入力中心になっていないか
- 変更範囲は1目的か
- 一時ファイルが残っていないか
- 秘密情報が含まれていないか
- 未完了をDONE扱いしていないか

## UI確認

- スマホ縦画面で確認したか
- 主要ボタンが押しやすいか
- 実画像マップが主役になっているか
- 田んぼ区画が分かるか
- 入水口、出水口、注意箇所の違いが分かるか
- 写真/音声記録への導線が明確か
- Preview URLまたはスクリーンショットがあるか

## Supabase確認

- migration適用前にSQL案をレビューしたか
- 既存テーブル削除が含まれていないか
- RLS方針がgroup単位か
- 未ログイン編集を許していないか
- Storageの公開範囲を確認したか

## GitHub確認

- main直pushしていないか
- 1PR 1目的か
- PR本文に変更内容と未確認事項があるか
- mergeはユーザー承認後か
- 大きすぎる変更が混ざっていないか

## DONE条件

- 変更内容が説明されている
- 確認方法が記録されている
- 未確認事項が明記されている
- 必要ならユーザー確認済み
