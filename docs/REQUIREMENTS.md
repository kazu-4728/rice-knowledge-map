# REQUIREMENTS.md

## 1. 目的

`rice-knowledge-map` は、実画像マップを中心に、稲作現場の暗黙知を記録・共有・継承するPWAです。

目的は、全国向けSaaSを最初から作ることではありません。まず自分の田んぼで使い、家族・作業者と場所と状態を共有できる最小実用版を作ります。

## 2. 解決する課題

稲作では、以下の情報が人に依存しやすいです。

- 入水口・出水口の位置
- 詰まりやすい場所
- 水が抜けにくい場所
- 雑草が出やすい場所
- 畦が崩れやすい場所
- 過去の異常と対応履歴
- 家族や先代しか知らない呼び名・注意点

このアプリは、これらを実画像マップ上に固定します。

## 3. 初期MVPの必須機能

### 3.1 実画像マップ

- MapLibreを使用する。
- 背景は国土地理院の空中写真タイルを第一候補にする。
- 将来、Google Maps satellite / Mapbox Satelliteに差し替え可能にする。
- 地図APIに業務データを依存させない。

### 3.2 田んぼ管理

- 田んぼ名
- 中心位置
- 区画ポリゴン
- 面積
- メモ
- 作期
- 表示順

### 3.3 固定ポイント管理

実画像マップ上に以下のポイントを登録できること。

- 入水口
- 出水口
- 水路
- 注意箇所
- 雑草箇所
- 畦崩れ箇所
- 水抜け不良箇所
- その他

### 3.4 記録作成

現場で1分以内に記録できることを優先する。

基本フロー:

1. 写真を撮る
2. 位置情報を取得する
3. 近い田んぼを推定する
4. 音声メモを追加する
5. AIで要約・分類する
6. 保存前確認を行う
7. 実画像マップ上に記録ピンを表示する

### 3.5 共有

- Google認証を前提にする。
- ユーザー個人ではなく、農場/作業グループ単位で共有する。
- 招待URLでグループ参加できるようにする。
- URLだけで未ログイン編集できる状態にはしない。

### 3.6 状態管理

記録には以下の状態を持たせる。

| UI表示 | DB値 | 意味 |
|---|---|---|
| 未対応 | open | まだ対応していない記録 |
| 要確認 | needs_check | 追加確認が必要な記録 |
| 対応済み | resolved | 対応が完了した記録 |
| 経過観察 | monitoring | 継続して様子を見る記録 |

## 4. 初期MVPで後回しにするもの

- 収量分析
- 売上管理
- 補助金帳票
- AI画像診断
- センサー連携
- 完全ハンズフリー音声入力
- 全国向けSaaS管理機能

## 5. 成功条件

- 実画像マップ上で田んぼとポイントが分かる。
- 写真と音声で記録できる。
- 記録が地図上に残る。
- 家族/作業者が場所を理解できる。
- 未対応箇所が分かる。
- スマホ現場利用で迷わない。

## 6. 非機能要件

- PWAとしてスマホ優先で使えること。
- 主要操作は片手で完結すること。
- Supabase RLSでグループ単位にアクセス制御すること。
- 秘密情報をGitHubに入れないこと。
- UI変更は段階的に行うこと。
