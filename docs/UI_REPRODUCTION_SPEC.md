# UI_REPRODUCTION_SPEC.md

## 1. 目的

画像で作成したUIの方向性を、実装画面として再現するための仕様です。

最初から完成アプリを作るのではなく、まずスマホで見て「現場で使えそうか」を判断できるPreviewを作ります。

## 2. UIの最重要方針

- スマホ縦画面を優先する。
- 実画像マップを主役にする。
- 手入力を主入力にしない。
- 写真・音声・位置情報を中心にする。
- 現場で1分以内に記録できることを優先する。
- 田んぼの位置・水口・異常箇所が直感的に分かることを優先する。

## 3. 初期Previewで再現する画面

### 3.1 ホーム

目的: 今日見るべき田んぼと未対応記録をすぐ確認する。

要素:

- アプリ名
- 今日の確認カード
- 未対応/要確認カード
- 写真で記録ボタン
- 音声で記録ボタン
- マップボタン
- 最近の記録

### 3.2 実画像マップ

目的: 田んぼの場所知識を実画像上で確認する。

要素:

- 国土地理院の空中写真タイル
- 田んぼポリゴン
- 田んぼ名ラベル
- 入水口ピン
- 出水口ピン
- 水路/注意箇所ピン
- 記録ピン
- 状態別色分け
- 下部詳細カード
- 「記録する」ボタン

### 3.3 田んぼ詳細

目的: 田んぼ単位で過去記録と固定ポイントを確認する。

要素:

- 田んぼ名
- 実画像マップ小表示
- 作期情報
- 固定ポイント一覧
- 最近の記録
- 未対応項目
- この田んぼで記録ボタン

### 3.4 写真で記録

目的: 現場で写真から記録を作る。

要素:

- 写真プレビュー
- 位置情報取得状態
- 田んぼ候補
- カテゴリチップ
- 地図サムネイル
- 位置を微調整
- 音声メモ追加
- 次へ

### 3.5 保存前確認

目的: AI整理結果を保存前に確認する。

要素:

- 写真サムネイル
- 田んぼ
- 地点
- カテゴリ
- 状態
- メモ
- 次の対応
- 文字起こし
- 保存ボタン

### 3.6 記録詳細

目的: 過去記録の状況とコメントを確認する。

要素:

- 写真
- 実画像マップ上の位置
- 状態
- 音声メモ
- 文字起こし
- 家族/作業者コメント
- 対応済みにする
- 追記する

## 4. 初期はダミーデータでよい

初期Previewでは、Supabase接続なしでよい。

理由:

- UI/UXを先に確認するため
- DB設計に引っ張られないため
- 実画像マップの使い勝手を早期に確認するため

## 5. 完了条件

UIタスクは以下がない限りDONEにしない。

- Preview URL またはスクリーンショット
- 対象画面名
- 確認した操作
- 未確認事項

## 6. 実装順

1. 実画像マップ画面
2. ダミー田んぼポリゴン
3. 入水口/出水口/注意箇所ピン
4. 下部詳細カード
5. 写真で記録画面
6. 音声メモ画面
7. 保存前確認画面
8. 記録詳細画面
