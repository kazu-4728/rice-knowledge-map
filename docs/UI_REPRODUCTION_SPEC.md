# UI_REPRODUCTION_SPEC.md

この文書は、参照モック画像の方向性をNext.js + Tailwind CSSで再現するための仕様です。

目的は、要素を並べることではありません。参照画像に近い、現代的なスマホアプリUIを作ることです。

---

## 1. UI判断の基準

参照モック画像をUI判断の基準にします。

文章で書かれた要素リストより、参照画像の見た目、余白、重なり、質感、画面密度、スマホアプリ感を優先してください。

すべての要素を1画面に詰め込む必要はありません。

見た目を悪くする要素は、小さくする、下部シートに入れる、タップ後に表示する、別画面へ移す、初期表示から外す、のいずれかで処理してください。

---

## 2. 目指すUI

このアプリは、農業管理表ではありません。実画像マップを中心にした現場ナレッジ記録アプリです。

目指す方向:

- スマホ縦画面
- フルスクリーンに近い実画像マップ
- 薄く軽いヘッダー
- 地図上の田んぼ区画
- 入水口、出水口、異常箇所のピン
- 下部ボトムシート
- フローティング操作ボタン
- 写真で記録、音声メモへの大きな導線
- 半透明パネル
- 角丸の大きいカード
- 柔らかい影
- 片手で押しやすいボタン
- 情報を詰め込みすぎない余白

避けたい方向:

- 古い業務システム風
- 管理ダッシュボード風
- 表や一覧が主役の画面
- 白いカードをただ並べるだけの画面
- 大きすぎるヘッダー
- 多すぎるフィルター
- ラベルだらけの地図
- 文字情報が多すぎる画面
- 天気カード、予定表、同期ステータスが主役に見える画面

---

## 3. メインマップ画面の基準

最初に品質を上げる画面は、メインマップ画面です。

初期表示で見えるべきもの:

- 実画像マップ
- 田んぼ区画
- 入水口、出水口、異常箇所のピン
- 現在選択中の地点を示す下部ボトムシート
- 写真で記録ボタン
- 音声メモボタン
- 最小限のヘッダー
- 最小限の下部ナビ

初期表示で主役にしないもの:

- 予定表
- 天気カード
- 同期状態
- データ出力
- 長い説明文
- 全件一覧
- フィルターの大量表示

---

## 4. Tailwind CSS方針

UIはTailwind CSSを標準にします。

- 余白、角丸、影、文字サイズをTailwindで統一する
- `globals.css` への大量追加は避ける
- 共通UIはコンポーネント化する
- 画面ごとに場当たり的なCSSを増やさない

---

## 5. 完了条件

UIタスクは、以下を満たすまでDONEにしないでください。

- スクリーンショットまたはPreview URLがある
- 参照モック画像と比較している
- 実画像マップが主役に見える
- 下部ボトムシートがスマホアプリらしく見える
- 写真・音声の記録導線が分かる
- 変更ファイルが説明されている
- まだ違う点が明記されている

---

## 6. 実装順

1. メインマップ画面を参照画像に近づける
2. App Shellを整える
3. ピン、区画、ボトムシートの見た目を整える
4. 写真で記録画面を作る
5. 音声メモ画面を作る
6. 保存前確認画面を作る
7. 記録詳細画面を作る
8. Supabase接続へ進む
