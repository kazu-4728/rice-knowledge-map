## Claude Code 固有ルール

- 作業開始時は、まず `tasks/TASKS.md` の「現在の実行タスク」を読む。
- 長時間のUI改修でも、現在タスクで定義された利用体験が完成するまで同じブランチ・同じDraft PRで進める。
- Draft解除は、受入条件を満たし、セルフレビュー完了後に行う。
- PR完了報告には、対応内容、修正コミット、検証結果、未確認事項を日本語で記載する。
- レビュー指摘を修正した場合は、該当スレッドへ返信し、解決済みならResolveする。
- squashマージ後は `tasks/TASKS.md` の現在タスク・次タスク・ユーザー確認待ちだけを更新する。
