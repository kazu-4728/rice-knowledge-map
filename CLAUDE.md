# CLAUDE.md

Claude Code向けの引き継ぎ文書です。

---

## 1. 現在の目的

目的は、実画像マップ型 稲作ナレッジ記録アプリを完成させることです。

テストモックは途中工程です。今は、最終アプリに使えるUI品質をNext.js内で作る段階です。

---

## 2. 最優先の作業

現在の最優先は、メインマップ画面を参照モック画像に近づけることです。

既存の要素リストを全部並べるのではなく、参照画像の構図、余白、質感、画面密度、スマホアプリ感を優先してください。

---

## 3. 目指すUI

優先するもの:

- フルスクリーンに近い実画像マップ
- 薄く軽いヘッダー
- 地図上の田んぼ区画とピン
- 下部ボトムシート
- フローティング操作ボタン
- 写真で記録、音声メモへの導線
- 片手で操作しやすいタップ領域
- 余白、角丸、影、半透明、重なりのあるスマホアプリUI

避けるもの:

- 古い業務システム風UI
- 管理ダッシュボード風UI
- 表や一覧が主役の画面
- 白いカードを並べるだけの画面
- 大きすぎるヘッダー
- 多すぎるフィルター
- ラベルだらけの地図
- 天気、予定、同期状態が主役に見える画面

---

## 4. 技術方針

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- MapLibre GL JS
- 国土地理院 空中写真タイル
- Supabase Auth / Postgres / Storage / RLS

UI確認もNext.js内で行います。HTMLだけの別モックは原則作りません。

`globals.css` は必要最小限にし、UIはTailwind CSSと共通コンポーネントで整えます。

---

## 5. 作業ルール

- UI作業とDB作業を混ぜない。
- UI作業と認証作業を混ぜない。
- UI作業と保存処理を混ぜない。
- 1タスク1目的にする。
- スクリーンショットまたはPreview URLなしでUI作業を完了扱いにしない。
- 参照モック画像との違いを報告する。

---

## 6. 次にやること

まず、メインマップ画面だけを基準品質まで上げてください。

見るべき点:

- 実画像マップが主役に見えるか
- 下部ボトムシートが自然か
- 写真・音声で記録する導線が強いか
- 画面全体がスマホアプリに見えるか
- 参照モック画像から離れていないか

この品質が合格するまで、DB接続や機能追加へ進まないでください。
